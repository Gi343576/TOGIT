
# 协作说明

## 背景

- 这是我的 Markdown 写作仓库，用于和 ChatGPT 协作推进自媒体故事项目
- 如果你是 ChatGPT，请先阅读本 README.md，再按本文列出的顺序读取相关文件，并以最新版内容为准
- 如果文件内容发生冲突，优先级如下：
    1. 本轮聊天中用户最新指令
    2. README.md 中的「当前任务」
    3. 排序靠前的项目文件
    4. 排序靠后的项目文件

## 我的文档：格式说明

- 文档使用 Obsidian 编写
- Markdown 语法符合通用规范
- 文件链接使用 Obsidian 的 `[[]]` 格式
- 你可以理解这些链接对应仓库中的同名 Markdown 文件

# 我对你的要求

## 你的输出规则

- 不要直接开始写完整正文，除非我明确要求
- 优先帮我搭结构、改分场、增强冲突链
- 修改时要说明：
    - 改了什么
    - 为什么改
    - 下一步需要我判断什么
- 尽量通过剧情、动作、画面和动物反应，让观众自己看出来
- 雪花不能只是好人，要突出他的弱点：太相信做事本身
- 黑猪不能只是坏人，要突出他懂动物的恐惧
- 风车必须作为核心成果贯穿故事

## 你的输出格式

- 输出标准的 Markdown 格式
- 用无序列表，代替正文
    - 无序列表的符号，用 `-`，而不是 `*`
- 不要分割线 `---`
- 缩进用 4 个空格
